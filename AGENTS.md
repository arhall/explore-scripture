# AGENTS.md

Quick, concise orientation for agents working in this repo. Use this before
diving into details in `CLAUDE.md`.

## Project Snapshot
- **Product**: Explore Scripture, a static Bible study platform.
- **Stack**: Eleventy (11ty) + Nunjucks + vanilla JS + CSS.
- **Deploy**: Cloudflare Pages; build output `_site/`.
- **Design**: Mobile-first, themeable via CSS custom properties.

## Fast Start (Local)
```bash
npm install
npm run dev
```
Other common commands:
```bash
npm run build
npm run build:analyze
npm run dev:pages
npm run test
npm run lint
```

## Core Structure (Know These Paths)
- **Layouts**: `src/_includes/layouts/`
  - `base.njk` full feature layout (nav, toggles, PWA, etc.)
  - `base-minimal.njk` simplified layout
  - `book.njk` book pages
  - `category.njk` category pages
- **Pages**: `src/*.njk` (home, books, categories, entities, map, links, etc.)
- **Data**: `src/_data/` (books, categories, videos, cross-refs, etc.)
- **Styles**: `src/styles.css` (large monolithic stylesheet)
- **Assets/JS**: `src/assets/` (interactive features, loaders, telemetry)

## High-Impact Features (Where to Look)
- **Chapter Reader**: `src/assets/chapter-reader.js`
- **Commentary System**: `src/assets/commentary-reader.js`
- **Search Engine/UI**: `src/assets/search-engine.js`,
  `src/assets/search-interface.js`
- **Scripture Widget**: `src/assets/scripture-widget.js`
- **Theme System**: `src/assets/theme-manager.js`
- **Map Page**:
  - Template: `src/map.njk`
  - JS: `src/assets/js/map/`
  - Map styles: `src/assets/map/styles/`
  - CSS: `src/styles.css` (map section near end)
- **Genealogies**: `src/genealogies.njk` (D3-based visualization)

## Data Updates (Common Tasks)
- **Book summaries/metadata**: `src/_data/books.json`
- **Categories**: `src/_data/categories.js`
- **Bible Project videos**: `src/_data/bibleProjectVideos.json`
- **Cross references**: `src/_data/crossReferences.js`
- **Entities**: generated via `scripts/entity-processor.js`

## Build & Data Generation
- Search data generated by `generate-search-data.js`.
- Build output goes to `_site/`.

## Architecture Notes (Short)
- Eleventy generates static pages from data + Nunjucks templates.
- JS is modular; many features load lazily via `src/assets/module-loader.js`
  and `src/assets/bundle-optimizer.js`.
- CSP is defined in `base.njk`; new external resources must be added there.

## Styling Conventions
- Use CSS variables; avoid hard-coded colors.
- Keep layouts mobile-first; test at narrow widths.
- Map page uses dedicated styles in `src/styles.css` (search for `.map-`).

## Testing
- JS tests: `npm run test`
- Selenium tests: `npm run test:selenium`
- iOS tests: `npm run test:ios`

## Deployment
- Cloudflare Pages; build command `npm run build`, output `_site/`.
